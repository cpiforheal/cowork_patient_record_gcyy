# 领导检查与验收指南

本文用于领导、科室负责人和管理人员在内测期间快速判断系统是否真正可用。

## 管理视角最关心的四个问题

1. 今天有没有必须处理的事项？
2. 哪些单据还没有闭环？
3. 哪些物资存在低库存、临期、过期、报废或盘点差异？
4. 数据能不能导出、复核、留档，并追溯到具体时间和人员？

## 每天 5 分钟检查

进入系统后优先查看：

1. `进销存管理 -> 进销存主控台`
   - 看待审核、待发放、待签收是否积压。
   - 看低库存、临期、过期、盘点差异和报废提示。
2. `进销存管理 -> 领导驾驶舱`
   - 看关键指标和异常趋势。
   - 重点关注红色或黄色提示。
3. `进销存管理 -> 追溯流水`
   - 抽查某个物资是否能查到入库、发放、签收、退回、报废或盘点记录。

如果每天只看一个页面，建议先看“进销存主控台”。它的目标是回答“今天谁还需要处理什么”。

## 每周检查

每周建议固定一次检查：

- 周消耗是否按科室填报。
- 本周申领是否都完成审核、发放和签收。
- 库存低于预警线的物资是否安排补充。
- 临期和过期物资是否有处理意见。
- 报废和盘点差异是否有原因说明。
- 是否需要导出 CSV 作为内部台账或会议材料。

## 判断闭环的标准

### 病历协同闭环

- 患者基础信息可查。
- 本岗位病历字段已保存。
- 必要附件已上传且能打开。
- 质控或审核意见已处理。
- 作废、恢复、异常修改有记录可查。

### 物资申领闭环

- 科室提交申领。
- 负责人完成审核。
- 库管完成发放并扣减库存。
- 科室完成签收。
- 后续周消耗、盘点、退回或报废有记录。

只要停留在待审核、待发放、待签收，就还没有闭环。

## 内测验收建议

第一轮内测建议选择 1 到 2 个科室先跑 2 周，重点观察：

- 一线是否愿意在系统里提交申领，而不是继续口头或微信申领。
- 库管是否能根据系统完成发放和追溯。
- 科室负责人是否能按系统待办完成审核。
- 领导是否能通过看板发现低库存、临期、报废和未闭环单据。
- 数据导出是否能满足内部检查和留档。

## 需要会议确认的管理规则

内测前建议由管理人员明确：

- 哪些岗位可以新建患者、修改病历、上传附件。
- 哪些人员负责物资建档、入库、发放和盘点。
- 科室负责人多久内完成申领审核。
- 发放后科室多久内必须签收。
- 周消耗固定由谁在每周哪一天填报。
- 报废、盘亏、过期物资由谁复核。
- 数据和附件由谁负责备份。

系统能把流程和记录固化下来，但内测是否顺利，很大程度取决于这些规则是否在上线前讲清楚。
