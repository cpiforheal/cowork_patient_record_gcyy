# 进销存模块 PRD、统计口径与岗位 SOP

> 文档版本：2.0  
> 适用版本：Flyway V12 及以上  
> 适用范围：中央仓库、科室库存、在途调拨、阶段耗用、库存异常与科室耗材报表

## 1. 产品目标与范围

本模块以“少录入、可追溯、账实可核对”为目标，将物资从中央仓库到科室、再到具体病历执行环节的流转统一到一套服务端账本中。

本版本覆盖：

- 物资档案、批次入库、库存批次和效期管理。
- 科室申领、审批锁定、发放在途、科室签收和科室退库。
- 科室套餐、执行阶段自动耗用、实际用量差异、失败重试和冲销补扣。
- 科室盘点、报损、库存流水、异常中心和周计划建议。
- 按科室查询及导出 A4 PDF、XLSX 耗材使用报表。

本版本不覆盖采购订单、供应商管理、采购付款、发票和财务凭证。相关需求不得通过库存流水临时代替。

## 2. 权限与数据边界

权限以服务端 `GET /auth/navigation` 下发的能力集合为唯一展示来源，前端不再自行维护角色写权限。后端对每个写接口继续强制鉴权。

| 角色 | 库存边界 |
| --- | --- |
| 管理员 | 系统配置、库存维护、异常处理、盘点确认、全部报表；应急越权操作必须审计 |
| 质控 | 库存控制、物资与套餐维护、异常处理、盘点确认、全部报表；不管理系统账号 |
| 管理者 | 库存与经营报表只读，不执行库存写操作 |
| 科室业务岗位 | 仅查看授权科室及完成本岗位的申领、签收、执行差异或退库动作 |
| 前台、临床及辅助岗位 | 不因角色名称获得跨科库存权限，数据范围由活动科室和病历归属科室决定 |

账号使用一个全局岗位角色，可关联多个稳定科室 ID。活动科室决定默认工作范围；病历创建后锁定 `owningDepartmentId`，库存耗用和报表归属均以该字段为准，不跟随操作人的当前科室变化。

科室名称只用于展示。科室改名不改变历史账目；已有引用的科室只能停用，不能物理删除。

## 3. 库存账本与统计口径

### 3.1 三类库位

- 中央仓库：入库、审批锁定、发放出库和退库接收的主体。
- 在途库位：中央仓库已发放但科室尚未签收的数量。
- 科室库存：科室签收后可被实际执行环节消耗的数量。

库存余额精确到“库位 + 科室（适用时）+ 物资 + 批次 + 有效期”。所有余额变化必须同时写入不可覆盖的库存流水。

### 3.2 流转规则

1. 入库：增加中央仓库库存。
2. 审批：只锁定中央仓库可用量，不改变账面数量。
3. 发放：中央仓库减少，在途增加；不计入科室耗用。
4. 签收：在途减少，科室库存增加；不计入科室耗用。
5. 执行耗用：从病历归属科室库存按 FEFO 扣减，计入实际耗用。
6. 退库：科室库存减少，中央仓库增加，单独列示。
7. 报损与盘点调整：作用于明确库位，单独列示，不混入耗用。
8. 阶段退回、病历作废或纠错：通过冲销及补扣流水修正，不删除原记录。

### 3.3 报表统一口径

科室实际耗用量定义为：

```text
实际耗用 = 执行耗用 - 耗用冲销
```

科室期末结存定义为：

```text
期末 = 期初 + 调入 - 退库 - 执行耗用 + 耗用冲销 - 报损 + 盘点调整
```

发放、签收、退库、报损和盘点必须分别列示。调拨不能重复计入耗用量。PDF、XLSX、页面汇总和底层流水必须使用同一服务端查询结果。

切换日前的历史数据标记为“历史推算”；切换日后的报表完全以 V12 库存流水为准。

## 4. 执行阶段自动耗用

### 4.1 套餐匹配

系统按以下维度选择生效套餐：

- 病历归属科室。
- 门诊/住院类型。
- 执行阶段：`INSPECTION`、`TCM`、`DOCTOR`、`SURGERY`。
- 套餐启用状态、生效日期和版本。

执行人员通常不需要逐项填写耗材。仅当实际用量与套餐不一致时登记增减差异，系统将套餐明细与差异合并为最终耗用清单。

### 4.2 幂等和一致性

阶段完成事务写入唯一耗用命令，持久化工作器异步执行：

- 同一病历、阶段和完成版本只能有一个有效耗用命令。
- 服务重启后未完成命令继续处理。
- 重复完成、双击和网络重试不会重复扣减。
- 物资按 FEFO 选择科室批次。
- 一个命令库存不足时不做部分扣减，避免半套成功。
- 成功耗用、批次明细和库存流水处于同一事务。

### 4.3 医疗流程与库存异常解耦

套餐缺失、科室期初未确认、库存不足或批次异常不阻断医疗阶段完成。系统创建可见异常任务，保存失败原因并提供重试入口。

阶段退回或病历作废时生成冲销命令；纠错影响用量时先冲销旧版本，再按新版本补扣。冲销保留原耗用关联、操作者、原因和时间。

## 5. 期初迁移和启用

Flyway V12 完成双账结构迁移：

- 旧批次数量作为中央仓库期初数保留。
- 根据历史发放、退库和自动耗用生成科室期初建议。
- 管理员或质控必须按实物盘点确认科室期初。
- 未确认科室不启用自动扣减，只创建异常任务。

首次启用顺序：

1. 核对物资档案、单位、批次和有效期。
2. 核对账号科室成员关系与病历归属科室。
3. 对中央仓库和各科室进行实物盘点。
4. 确认科室期初建议。
5. 配置并启用各执行阶段套餐版本。
6. 用测试病历验证完成、重复完成、退回冲销和异常重试。
7. 对照库存流水、科室余额和报表后再正式启用。

## 6. 界面与旧入口兼容

管理工作台收束为四个视觉中心：

1. 业务流转：入库 → 申领审批 → 配发/在途 → 科室签收。
2. 科室库存：当前结存 → 执行耗用 → 退库/盘点。
3. 异常中心：缺货、套餐缺失、失败耗用、待冲销和临期库存。
4. 管理分析：科室耗用、库存风险、周计划和纸质报表。

页面以状态数量、流程节点、待办徽标和主操作按钮为视觉重心，不使用大段说明文字。岗位进入后默认聚焦下一项待办。

以下原有入口继续兼容并映射到新工作台区域：

- `/inventory/overview`
- `/inventory/executive`
- `/inventory/requests`
- `/inventory/stock`
- `/inventory/items`
- `/inventory/weekly`
- `/inventory/controls`
- `/inventory/packages`
- `/inventory/trace`

`/inventory/manage` 继续作为总入口，旧收藏和链接不失效。

## 7. 核心接口

### 7.1 工作台与账本查询

- `GET /inventory-api/workbench`
- `GET /inventory-api/department-balances`
- `GET /inventory-api/exception-tasks`
- `GET /inventory-api/consumption-events`
- `POST /inventory-api/department-openings/confirm`
- `POST /inventory-api/exception-tasks/retry`

### 7.2 报表

- `GET /inventory-api/reports/department-usage`
- `GET /inventory-api/reports/department-usage.pdf`
- `GET /inventory-api/reports/department-usage.xlsx`

查询支持日期范围、一个或多个科室、物资分类、物资和执行环节。报表查询和导出写入审计日志。

PDF 为 A4 纸质归档版本，每个科室另起一页，包含期初、调入、退库、实际耗用、冲销、报损、调整和期末结存，并附脱敏病历明细。XLSX 包含全院汇总页和每科室独立工作表，统计口径与 PDF 一致。

明细只展示日期、就诊流水号、脱敏患者、执行环节、套餐版本、物资批次、数量和操作来源，不输出完整身份证、电话或地址。

### 7.3 兼容接口

原物资、入库、申领、审批、发放、签收、退库/报损、盘点、套餐和旧耗用查询接口保持可用。新流程在服务端将相关动作转换为调拨或账本事务，调用方不得直接修改余额。

## 8. 岗位 SOP

### 8.1 中央仓库/质控

1. 在“业务流转”查看待审批申请及中央仓库可用量。
2. 审批后由系统锁定数量；缺货时调整或驳回，不手工伪造库存。
3. 配发时核对批次和有效期，确认后库存进入在途。
4. 对逾期未签收、临期和失败耗用在“异常中心”处理。
5. 科室退库到仓后完成反向调拨确认。

### 8.2 科室收货岗位

1. 在本活动科室查看在途单。
2. 按实物核对物资、批次、数量和效期。
3. 一致时签收，库存进入科室库；不一致时记录差异并停止错误签收。

### 8.3 临床及辅助执行岗位

1. 正常完成本岗位执行阶段，无需重复录入套餐耗材。
2. 实际用量不一致时只登记差异及原因。
3. 阶段完成不因库存异常被阻断；出现异常提示后由库存管理岗位处理。
4. 需要退回或纠错时走原业务流程，禁止删除或直接改动库存流水。

### 8.4 管理员/质控异常处理

1. 查看异常类型、病历归属科室、阶段、套餐版本和失败原因。
2. 先修复期初确认、套餐、批次或库存问题。
3. 使用“重试”继续原幂等命令，不新建人工耗用绕过异常。
4. 对退回、作废和纠错检查冲销/补扣是否完成。

### 8.5 管理者报表

1. 选择日期和科室范围，先查看页面汇总。
2. 核对期初、流转、实际耗用、冲销和期末守恒。
3. 单科室归档可直接导出 PDF；跨科分析导出 XLSX。
4. 纸质签字归档按科室分页保存，不另行手工重算。

## 9. 周计划

周计划建议量由实际耗用、当前科室库存和安全库存自动计算。人工只填写异常说明和最终调整量，不再重复录入“本周已耗用”。

建议值用于计划，不直接创建入库或财务采购单。最终调整必须保留修改人、时间和原因。

## 10. 验收与对账

上线前至少覆盖以下场景：

- 入库、审批锁定、发放在途、签收科室库存全链路守恒。
- 审批不扣账、发放不计耗用、签收不重复增加中央库存。
- FEFO 扣减、多批次耗用、重复完成、并发完成和服务重启恢复。
- 套餐缺失、期初未确认、库存不足均只创建异常且不部分扣减。
- 阶段退回、作废、纠错形成冲销及补扣，原流水仍可追溯。
- 退库、报损和盘点作用于正确库位且分别统计。
- 多科室切换不改变病历锁定归属，科室改名不改变历史统计。
- 管理者只读，普通岗位越权库存写请求被后端拒绝。
- PDF、XLSX、页面合计、库存流水和科室余额一致。
- 报表病历明细脱敏，每科室 PDF 独立分页，导出操作进入审计。

上线门禁：Flyway V11/V12 在真实 MySQL 升级成功、后端测试和前端校验通过、原九个路由可解析、8848 健康检查通过、数据库/附件/部署包备份可恢复。

## 11. 故障与回滚

### 11.1 自动耗用集中失败

暂停异常任务重试，先判断是否为科室期初、套餐版本、批次或库存不足。医疗业务可继续；修复根因后按原命令重试，禁止直接删除失败命令。

### 11.2 疑似重复扣减

立即停止相关阶段的套餐启用，按“病历 + 阶段 + 完成版本”核对命令、耗用事件和库存流水。通过冲销修正，不直接改数据库余额。

### 11.3 发布回滚

发布前备份数据库、附件和当前交付包。若迁移或启动失败，停止当前 8848 实例，恢复匹配版本的数据库和交付包后仍在 8848 启动。禁止只回滚程序而保留不兼容数据库，也禁止另开端口规避失败。
